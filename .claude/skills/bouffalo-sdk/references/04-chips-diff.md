# 芯片系列差异

## 芯片系列概览

Bouffalo SDK 支持多个芯片系列，每个系列针对不同的应用场景，在性能、功耗、外设等方面各有侧重。

| 芯片系列 | 核心数 | 架构 | 主频 | 无线协议 | 应用场景 |
|---------|-------|------|------|---------|---------|
| BL602/BL604 | 单核 | RISC-V | 320MHz | WiFi 4, BLE | 低功耗IoT |
| BL702/BL704/BL706 | 单核 | RISC-V | 320MHz | BLE | 可穿戴设备 |
| BL616/BL618 | 双核 | RISC-V (M0+LP) | 320MHz | WiFi 6, BLE | 高性能IoT |
| BL616D | 双核 | RISC-V (M0+LP) | 320MHz | WiFi 6, BLE | 高性能IoT |
| BL616L | 双核 | RISC-V (M0+LP) | 320MHz | BLE | 可穿戴设备 |
| BL808 | 多核 | RISC-V (M0+D0) | 320MHz | WiFi, 丰富外设 | 智能家电 |
| BL606P | 多核 | RISC-V (M0+D0) | 320MHz | WiFi, 丰富外设 | 工业IoT |

## 核心差异

### BL602/BL604 系列

**特点：**
- 单核 RISC-V (320MHz)
- WiFi 4 + BLE
- 低功耗设计
- 基础外设支持

**适用场景：**
- 智能家居传感器
- 低功耗IoT设备
- WiFi + BLE双模设备

**限制：**
- 外设支持较少（GPIO/UART等部分未充分测试）
- 无DMA（未充分测试）
- 安全外设未充分测试

### BL702/BL704/BL706 系列

**特点：**
- 单核 RISC-V (320MHz)
- BLE专用
- 丰富外设
- 成本优化

**适用场景：**
- 可穿戴设备（手环、手表）
- BLE传感器
- 便携设备

**优势：**
- 完整的外设支持
- USB v1支持
- 摄像头接口

**限制：**
- 仅支持BLE
- 无WiFi

### BL616/BL618 系列

**特点：**
- 双核 (M0 + LP)
- WiFi 6 + BLE
- 高性能外设
- USB v2

**适用场景：**
- 高性能IoT网关
- 智能摄像头
- WiFi 6设备

**优势：**
- WiFi 6支持
- 完整的外设支持
- USB v2
- 双核架构（LP核心用于低功耗）

**特别：**
- BL616D：类似BL616，针对特定市场
- BL616L：仅BLE，无WiFi

### BL808 系列

**特点：**
- 多核 (M0 + D0)
- WiFi（版本待确认）
- 丰富外设
- 高性能

**适用场景：**
- 智能家电
- 工业IoT
- 需要高性能的应用

**优势：**
- 多核架构（D0核心可能用于加速）
- CSI接口（摄像头）
- 最丰富的外设支持

**限制：**
- BLE支持（标记为×）
- Zigbee/Thread支持（标记为×）

## 编译差异

### 单核芯片（BL602、BL702、BL616、BL618）

```bash
make CHIP=bl616 BOARD=bl616dk
```

### 多核芯片（BL808、BL606P、BL616D）

需要指定 `CPU_ID`：

```bash
# M0核心（主控）
make CHIP=bl808 BOARD=bl808dk CPU_ID=m0

# D0核心（专用，如WiFi）
make CHIP=bl808 BOARD=bl808dk CPU_ID=d0
```

### BL616L（仅BLE）

```bash
make CHIP=bl616l BOARD=bl616ldk
```

## SDK配置差异

### WiFi配置

**BL602 (WiFi 4):**
```bash
# Kconfig
CONFIG_WIFI4=y
```

**BL616 (WiFi 6):**
```bash
# Kconfig
CONFIG_WIFI6=y
```

**BL808:**
```bash
# Kconfig
# WiFi配置（具体版本待确认）
```

### BLE配置

**所有支持BLE的芯片：**
```bash
# Kconfig
CONFIG_BLE=y
```

### 双核配置（BL808、BL606P、BL616D）

```bash
# M0核心
CONFIG_CORE_M0=y

# D0核心
CONFIG_CORE_D0=y
```

### LP核心配置（BL616、BL616D、BL616L）

```bash
# LP核心
CONFIG_CORE_LP=y
```

## 选择建议

### 低功耗IoT（传感器）
- **推荐**: BL602
- **理由**: WiFi 4 + BLE，低功耗，成本低

### 可穿戴设备（手环、手表）
- **推荐**: BL702 或 BL616L
- **理由**: BLE，丰富外设，成本优化

### 高性能IoT网关
- **推荐**: BL616 或 BL808
- **理由**: WiFi 6（BL616）或高性能（BL808），丰富外设

### 智能家电
- **推荐**: BL808
- **理由**: 多核，高性能，丰富外设

### 智能摄像头
- **推荐**: BL616 或 BL808
- **理由**: 摄像头接口，高性能

## 移植建议

### 从BL602到BL616

**相同点：**
- LHAL API相同
- WiFi支持（但版本不同）
- BLE支持

**不同点：**
- WiFi 4 → WiFi 6
- 单核 → 双核（M0 + LP）
- 外设支持更完整

**迁移步骤：**
1. 更新 `CHIP=bl616`
2. 检查WiFi配置（WiFi 4 → WiFi 6）
3. 检查LP核心使用（可选）
4. 测试外设（部分BL602未测试的外设在BL616已验证）

### 从BL702到BL616

**相同点：**
- LHAL API相同
- BLE支持
- 外设接口

**不同点：**
- BLE → WiFi 6 + BLE
- 单核 → 双核（M0 + LP）
- 新增WiFi功能

**迁移步骤：**
1. 更新 `CHIP=bl616`
2. 启用WiFi配置
3. 检查双核配置
4. 测试WiFi功能

### 从BL616到BL808

**相同点：**
- LHAL API相同
- WiFi支持（版本可能不同）
- 外设接口

**不同点：**
- 双核（M0 + LP）→ 多核（M0 + D0）
- 更大Flash/SRAM
- 更多外设（CSI等）

**迁移步骤：**
1. 更新 `CHIP=bl808`
2. 指定 `CPU_ID=m0` 或 `CPU_ID=d0`
3. 适配内存配置
4. 测试多核通信（如需要）
5. 测试新增外设

## 总结

Bouffalo SDK支持多个芯片系列，通过LHAL统一API，应用代码可以在不同芯片间无缝移植。选择芯片时，需根据应用场景、性能需求、功耗要求、成本等因素综合考虑。移植时，主要关注芯片特有的功能（如WiFi版本、核心架构、外设支持）。
